# Custom Base64 Encoding

## Overview

Qoder CLI uses a custom base64 encoding scheme for the `info` field in the JWT payload. The standard base64 alphabet is replaced with a shuffled 64-character alphabet and a custom padding character.

## Alphabet

Found in binary at VA offset `0x20d5720`:

```
_doRTgHZBKcGVjlvpC,@aFSx#DPuNJme&i*MzLOEn)sUrthbf%Y^w.(kIQyXqWA!
```

This is a direct 1:1 substitution of the standard Base64 alphabet:

```
Standard: ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/
Custom:   _doRTgHZBKcGVjlvpC,@aFSx#DPuNJme&i*MzLOEn)sUrthbf%Y^w.(kIQyXqWA!
```

## Padding Character

Found in binary at VA `0x1020d0988` (value `36` = ASCII `$`):

```
Padding: $
```

This replaces the standard `=` padding character.

## Implementation Functions

| Function | VA Range | Purpose |
|----------|----------|---------|
| `encrypt.(*encoding).encodeToString` | `0x10037f7c0` - `0x10037fa30` | Encode bytes to custom base64 string |
| `encrypt.(*encoding).encodeTo` | `0x10037fa30` - `0x10037fd20` | Encode bytes to custom base64 (buffer version) |
| `encrypt.init` | `0x10037f620` - `0x10037f7c0` | Initialize encoding table at startup |

## Encoding Algorithm

1. Input bytes → standard base64 encode
2. For each character in the standard base64 output:
   - If it's in the standard alphabet (`A-Za-z0-9+/`), replace with the corresponding character from the custom alphabet
   - If it's `=`, replace with `$`
3. Output the substituted string

## Decoding Algorithm

1. For each character in the custom base64 input:
   - If it's in the custom alphabet, find its index and replace with the corresponding standard base64 character
   - If it's `$`, replace with `=`
2. Standard base64 decode the result

## Python Implementation

```python
import base64

CUSTOM_ALPHABET = '_doRTgHZBKcGVjlvpC,@aFSx#DPuNJme&i*MzLOEn)sUrthbf%Y^w.(kIQyXqWA!'
STANDARD_BASE64 = 'ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/'
PADDING_CUSTOM = '$'
PADDING_STANDARD = '='

def custom_b64_encode(data: bytes) -> str:
    """Encode bytes using Qoder's custom base64."""
    std = base64.b64encode(data).decode('ascii')
    result = []
    for c in std:
        if c == '=':
            result.append(PADDING_CUSTOM)
        elif c in STANDARD_BASE64:
            result.append(CUSTOM_ALPHABET[STANDARD_BASE64.index(c)])
        else:
            result.append(c)  # Keep unknown chars as-is
    return ''.join(result)

def custom_b64_decode(encoded: str) -> bytes:
    """Decode Qoder's custom base64 to bytes."""
    std = []
    for c in encoded:
        if c == PADDING_CUSTOM:
            std.append(PADDING_STANDARD)
        elif c in CUSTOM_ALPHABET:
            std.append(STANDARD_BASE64[CUSTOM_ALPHABET.index(c)])
        else:
            std.append(c)  # Keep unknown chars as-is
    return base64.b64decode(''.join(std))
```

## TypeScript Implementation

```typescript
const CUSTOM_ALPHABET = '_doRTgHZBKcGVjlvpC,@aFSx#DPuNJme&i*MzLOEn)sUrthbf%Y^w.(kIQyXqWA!';
const STANDARD_BASE64 = 'ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/';
const PADDING_CUSTOM = '$';
const PADDING_STANDARD = '=';

function customBase64Encode(data: Buffer): string {
    const std = data.toString('base64');
    let result = '';
    for (const c of std) {
        if (c === '=') {
            result += PADDING_CUSTOM;
        } else {
            const idx = STANDARD_BASE64.indexOf(c);
            result += idx >= 0 ? CUSTOM_ALPHABET[idx] : c;
        }
    }
    return result;
}

function customBase64Decode(encoded: string): Buffer {
    let std = '';
    for (const c of encoded) {
        if (c === PADDING_CUSTOM) {
            std += PADDING_STANDARD;
        } else {
            const idx = CUSTOM_ALPHABET.indexOf(c);
            std += idx >= 0 ? STANDARD_BASE64[idx] : c;
        }
    }
    return Buffer.from(std, 'base64');
}
```

## Length Analysis for `info` Field

| Step | Size | Notes |
|------|------|-------|
| Random hex input | 16 bytes (32 hex chars) | Generated by `uuid` or `crypto/rand` |
| RSA-PKCS1v15 output | 128 bytes | 1024-bit key / 8 |
| ??? intermediate step | 252 bytes | **Unknown transformation** |
| Custom base64 encode | 336 chars | 252 * 4/3 = 336 (with `$` padding) |

The gap between 128 and 252 bytes is the only unexplained step. See `docs/rsa-key-discovery.md` for investigation.

## Frida Hook Confirmation

Frida successfully hooked `RsaEncrypt` and captured:
- **Input**: 16-byte random hex (plaintext)
- **Output**: 128 bytes (RSA-1024 ciphertext)
- **Public key**: Full PEM (see `docs/rsa-key-discovery.md`)

However, Frida **crashes** when attempting to read memory in the Go process due to signal handler conflicts (`SA_ONSTACK` not set for non-Go code). This prevents direct inspection of the intermediate 128→252 byte transformation.
